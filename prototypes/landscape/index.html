橫向專屬版面 · 同一份 CSS、四種螢幕

鎖手機橫放後，所有目標尺寸（手機 667×375 ~ 桌面 1920×1080）長寬比都在 1.6 ~ 2.1 之間，這時候可以用同一套 Grid + vh + clamp()，不用斷點切換。 下面四個畫面是同一個 HTML 檔案在不同 iframe 尺寸下的實際渲染。

核心設計原則

① 高度為基準 所有 padding / font / 元件高度都用 vh 為主、vw 為輔，因為橫向時高度是瓶頸。
② Grid 三列鎖死 grid-template-rows: auto 1fr auto，info-bar / 遊戲區 / 計時條，永遠對齊，不會擠壓。
③ clamp() 控字體 font-size: clamp(.85rem, 2.2vh, 1.35rem) — 手機橫放 ~14px、桌面 ~22px，自動縮放。
④ 帽子退居角落 帽子 absolute 定位在題目區右下角，不再搶橫向空間；教練泡泡浮在題目下緣全寬居中。
⑤ Duel 用 3 欄共題 P1 選項 ｜ 共用題目 ｜ P2 選項，題目只渲染一次，HP+分數移到 info-bar。
⑥ 直向跳遮罩 @media (orientation: portrait) 顯示「請橫放」動畫，不勉強縮塞。
