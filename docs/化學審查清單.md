## 1. 「易混淆」(`confused`) 類別已拆掉 —— 請確認重新分類

原本 5 個苯環衍生物被籠統歸成「易混淆」，學生答題時正確答案竟然是「易混淆」三個字，不合理。已改為各自真正的官能基分類：

| 分子 | 原分類 | 改成 | 開發團隊的理解（請確認）|
|---|---|---|---|
| 苯酚 phenol | 易混淆 | **酚 (phenol，自成一類)** | -OH 接在苯環 sp² 碳上，酸性 / 反應性與醇不同 → 自成一類（與 Level 99 的 "Phenol / Derivative" 一致）|
| 苯甲醇 benzyl alcohol | 易混淆 | 醇 alcohol | -CH₂-OH，OH 接在 sp³ 碳 → 正統脂肪醇 |
| 苯甲醛 benzaldehyde | 易混淆 | 醛 aldehyde | -CHO |
| 苯甲酸 benzoic acid | 易混淆 | 羧酸 carboxylic | -COOH |
| 苯胺 aniline | 易混淆 | 胺 amine | -NH₂（芳香胺，但仍屬胺類）|

**待老師確認**：
- [ ] 以上分類 OK 嗎？
- [ ] 「酚自成一類」這個處理 OK 嗎？（有些教材把酚當醇的特例，有些獨立講。本遊戲採「獨立一類」）

> 程式上：圖檔仍放在 `assets/images/13_confused/` 資料夾（資料夾名沒改，不影響分類），只改了 `data(organic).js` 裡的 `category` 欄位。

## 2. Level 6 改名「綜合題」

Level 6 原本叫「終極分類帽 (易混淆)」，但它的實際內容是「Level 1~5 的混合題」，「易混淆」用詞不準。已改成「終極分類帽 (綜合題)」。**這純粹是 UI 文字，不涉及化學。**

## 3. 答錯時的「為什麼」提示文案（草稿，待老師潤飾）⭐

規劃在練習模式答錯時，除了高亮正確答案，還顯示一句「辨識重點」。以下是**草稿**，請老師改成最精準、最適合學生的說法：

| 官能基 | 草稿文案 |
|---|---|
| 烷 alkane | 只有 C–C 單鍵和 C–H，沒有任何含氧 / 氮 / 鹵的官能基 |
| 烯 alkene | 含有 C=C 雙鍵 |
| 炔 alkyne | 含有 C≡C 參鍵 |
| 醇 alcohol | –OH 接在飽和（sp³）碳上 |
| 醚 ether | C–O–C，氧夾在兩個碳之間（沒有 OH、也沒有 C=O）|
| 醛 aldehyde | –CHO：羰基 C=O 在碳鏈末端，旁邊接一個 H |
| 酮 ketone | 羰基 C=O 夾在兩個碳之間 |
| 羧酸 carboxylic | –COOH：同一個碳上同時有 C=O 和 –OH |
| 酯 ester | –COO–：像羧酸但 –OH 被換成 –O–碳（C(=O)–O–C）|
| 胺 amine | 含 –NH₂ / –NHR / –NR₂（氮接碳，沒有羰基）|
| 芳香烴 aromatic | 含苯環，且苯環上只接烷基 / 乙烯基等碳氫基（沒有其他官能基）|
| 鹵化物 halide | 含 C–X（X = F、Cl、Br、I）|
| 酚 phenol | –OH **直接接在苯環的碳**上（若接 –CH₂– 之類則算醇，不是酚）|

**待老師確認**：
- [ ] 每句話的措辭是否精準、是否符合教學用語？
- [ ] 要不要再加一句「常見陷阱」？例如「苯甲醇是醇不是酚」「甲酸甲酯是酯不是酸」

## 4. 干擾選項的「相似度」邏輯（程式已用「同關卡優先」，未來可細化）

目前程式做法：答題時的 3 個錯誤選項，**優先從「同一個 Level 的分子」抽**（因為同關卡的官能基通常較接近、較有鑑別度），不夠再從前面學過的關卡補。這不需要化學判斷就能運作。

**未來若想更精準**，可以給一張「哪些官能基互為易混淆」的對照表（草稿如下，待老師審）：

| 官能基 | 容易跟它搞混的 |
|---|---|
| 醛 | 酮、羧酸 |
| 酮 | 醛、酯 |
| 羧酸 | 酯、醛、醇 |
| 酯 | 羧酸、酮、醚 |
| 醇 | 醚、酚、羧酸 |
| 醚 | 醇、酯 |
| 酚 | 醇、芳香烴、醚 |
| 烷 | 烯、炔、鹵化物 |
| 烯 | 烷、炔、芳香烴 |
| 胺 | 醇、鹵化物 |
| 鹵化物 | 烷、醇、芳香烴 |
| 芳香烴 | 烯、酚、鹵化物 |

- [ ] 這張表合理嗎？要不要這個細化（不做也能玩，只是干擾選項會比較「隨關卡」而非「隨官能基相似度」）
