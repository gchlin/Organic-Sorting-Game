## 1. 「易混淆」(`confused`) 類別已拆掉 —— 請確認重新分類

原本 5 個苯環衍生物被籠統歸成「易混淆」，學生答題時正確答案竟然是「易混淆」三個字，不合理。已改為各自真正的官能基分類：

| 分子 | 原分類 | 改成 | 開發團隊的理解（請確認）|
|---|---|---|---|
| 苯酚 phenol | 易混淆 | **酚 (phenol，自成一類)** | -OH 接在苯環 sp² 碳上，酸性 / 反應性與醇不同 → 自成一類（與 Level 99 的 "Phenol / Derivative" 一致）|
| 苯甲醇 benzyl alcohol | 易混淆 | 醇 alcohol | -CH₂-OH，OH 接在 sp³ 碳 → 正統脂肪醇 |
| 苯甲醛 benzaldehyde | 易混淆 | 醛 aldehyde | -CHO |
| 苯甲酸 benzoic acid | 易混淆 | 羧酸 carboxylic | -COOH |
| 苯胺 aniline | 易混淆 | 胺 amine | -NH₂（芳香胺，但仍屬胺類）|

**待老師確認**：
- [ ] 以上分類 OK 嗎？
- [ ] 「酚自成一類」這個處理 OK 嗎？（有些教材把酚當醇的特例，有些獨立講。本遊戲採「獨立一類」）

> 程式上：圖檔仍放在 `assets/images/13_confused/` 資料夾（資料夾名沒改，不影響分類），只改了 `data(organic).js` 裡的 `category` 欄位。

## 2. Level 6 改名「綜合題」

Level 6 原本叫「終極分類帽 (易混淆)」，但它的實際內容是「Level 1~5 的混合題」，「易混淆」用詞不準。已改成「終極分類帽 (綜合題)」。**這純粹是 UI 文字，不涉及化學。**

## 3. 答錯時的「為什麼」提示文案（草稿，待老師潤飾）⭐

規劃在練習模式答錯時，除了高亮正確答案，還顯示一句「辨識重點」。以下是**草稿**，請老師改成最精準、最適合學生的說法：

| 官能基 | 草稿文案 |
|---|---|
| 烷 alkane | 只有 C–C 單鍵和 C–H，沒有任何含氧 / 氮 / 鹵的官能基 |
| 烯 alkene | 含有 C=C 雙鍵 |
| 炔 alkyne | 含有 C≡C 參鍵 |
| 醇 alcohol | –OH 接在飽和（sp³）碳上 |
| 醚 ether | C–O–C，氧夾在兩個碳之間（沒有 OH、也沒有 C=O）|
| 醛 aldehyde | –CHO：羰基 C=O 在碳鏈末端，旁邊接一個 H |
| 酮 ketone | 羰基 C=O 夾在兩個碳之間 |
| 羧酸 carboxylic | –COOH：同一個碳上同時有 C=O 和 –OH |
| 酯 ester | –COO–：像羧酸但 –OH 被換成 –O–碳（C(=O)–O–C）|
| 胺 amine | 含 –NH₂ / –NHR / –NR₂（氮接碳，沒有羰基）|
| 芳香烴 aromatic | 含苯環，且苯環上只接烷基 / 乙烯基等碳氫基（沒有其他官能基）|
| 鹵化物 halide | 含 C–X（X = F、Cl、Br、I）|
| 酚 phenol | –OH **直接接在苯環的碳**上（若接 –CH₂– 之類則算醇，不是酚）|

**待老師確認**：
- [ ] 每句話的措辭是否精準、是否符合教學用語？
- [ ] 要不要再加一句「常見陷阱」？例如「苯甲醇是醇不是酚」「甲酸甲酯是酯不是酸」

## 4. 干擾選項的「相似度」邏輯（程式已用「同關卡優先」，未來可細化）

目前程式做法：答題時的 3 個錯誤選項，**優先從「同一個 Level 的分子」抽**（因為同關卡的官能基通常較接近、較有鑑別度），不夠再從前面學過的關卡補。這不需要化學判斷就能運作。

**未來若想更精準**，可以給一張「哪些官能基互為易混淆」的對照表（草稿如下，待老師審）：

| 官能基 | 容易跟它搞混的 |
|---|---|
| 醛 | 酮、羧酸 |
| 酮 | 醛、酯 |
| 羧酸 | 酯、醛、醇 |
| 酯 | 羧酸、酮、醚 |
| 醇 | 醚、酚、羧酸 |
| 醚 | 醇、酯 |
| 酚 | 醇、芳香烴、醚 |
| 烷 | 烯、炔、鹵化物 |
| 烯 | 烷、炔、芳香烴 |
| 胺 | 醇、鹵化物 |
| 鹵化物 | 烷、醇、芳香烴 |
| 芳香烴 | 烯、酚、鹵化物 |

- [ ] 這張表合理嗎？要不要這個細化（不做也能玩，只是干擾選項會比較「隨關卡」而非「隨官能基相似度」）

## 7.「其他（暫未收錄的官能基）」桶 —— 新增分子待審 ⭐（2026-05-12）

教學主軸維持 13 類；但段考會出、不在這 13 類內的**單官能基**化合物，我們打算統一歸到一個「**其他（暫未收錄的官能基）**」桶（學生答這類分子時選「其他」，圖鑑卡片再說明它真正是哪一族、課本還沒教到）。**多官能基的複雜藥物分子（阿斯巴甜、紫杉醇、阿司匹靈、柳酸、魯米諾、多巴胺…）一律不收。**

已用 RDKit（跟現有分子同樣的畫法：顯示所有 C、加上所有 H、元素配色）產出以下 15 個 SVG，放在 `assets/images/15_amide/` … `19_acyl_chloride/`，**目前只是圖檔、還沒登錄進 `data(organic).js`**：

| 族（資料夾）| 分子 | SMILES | 開發團隊的理解（請確認）|
|---|---|---|---|
| 醯胺 amide (`15_amide`) | 乙醯胺、丙醯胺 | `CC(N)=O`、`CCC(N)=O` | 一級醯胺 –CO–NH₂ |
| | N-甲基乙醯胺 | `CNC(C)=O` | 二級醯胺 –CO–NH–（蛋白質「肽鍵」的模型）|
| | 尿素 | `NC(N)=O` | 課本常見；兩個 –NH₂ 接同一個 C=O（碳醯胺）|
| | 苯甲醯胺 | `NC(=O)c1ccccc1` | 芳香環上的醯胺 |
| 硝基化合物 nitro (`16_nitro`) | 硝基甲烷、硝基乙烷 | `C[N+](=O)[O-]`、`CC[N+](=O)[O-]` | –NO₂ 接碳 |
| | 硝基苯 | `[O-][N+](=O)c1ccccc1` | 苯環上的 –NO₂ |
| 腈 nitrile (`17_nitrile`) | 乙腈、丙腈 | `CC#N`、`CCC#N` | –C≡N |
| | 苯甲腈 | `N#Cc1ccccc1` | 苯環上的 –C≡N |
| 酸酐 anhydride (`18_anhydride`) | 乙酸酐、丙酸酐 | `CC(=O)OC(C)=O`、`CCC(=O)OC(=O)CC` | –CO–O–CO– |
| 醯氯 acyl chloride (`19_acyl_chloride`) | 乙醯氯、苯甲醯氯 | `CC(Cl)=O`、`O=C(Cl)c1ccccc1` | –CO–Cl |

**待老師確認（重點）**：

- [ ] **硝基（–NO₂）的畫法**：RDKit 預設把 –NO₂ 畫成帶**形式電荷**的樣子（氮上一個正號 N⁺、其中一個氧上一個負號 O⁻、另一個 N=O 雙鍵）。這對高中生會不會造成困擾？課本通常怎麼畫硝基（純 N(=O)=O？還是也帶電荷的共振式？）—— 若課本畫法不同，我們可以調整產圖方式或這幾個分子直接不收。
- [ ] **酸酐（–CO–O–CO–）的結構辨識度**：乙酸酐 / 丙酸酐 的結構式畫出來，學生有沒有辦法跟「酯」區分清楚？酸酐在高中段考的出現頻率值不值得放進來？（開發團隊傾向：若意義不大就先不收酸酐、醯氯這兩族。）
- [ ] **「其他」桶的範圍**：上面 5 族（醯胺 / 硝基 / 腈 / 酸酐 / 醯氯）哪些該收、哪些別收？哪些分子是好例子、哪些換掉？
- [ ] **歸類方式**：把這些分子的「正確答案」都設成「其他」（而不是分別叫「醯胺」「硝基」），這樣處理在教學上 OK 嗎？（理由：避免在還沒正式教這些族的情況下，要求學生選出族名；圖鑑卡片仍會註明它真正屬於哪一族。）
- [ ] 這些分子的中文命名（乙醯胺 / N-甲基乙醯胺 / 硝基甲烷 / 乙腈 / 乙酸酐 / 乙醯氯…）用詞是否符合教學慣例？

> 程式上：確認後只需在 `data(organic).js` 的 `AnswerBank` 加上 `category: "other"`（或老師指定的分類值）、加一個「其他」答案選項、再把要用的分子編進 `QuestionSets`（哪一關 / 要不要獨立一關待定）。圖檔已就緒。`prototypes/test.html` 已加一段預覽可目視檢查這 15 張 SVG。
