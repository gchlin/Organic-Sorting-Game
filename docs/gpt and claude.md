# GPT vs Claude 遊戲評估討論

## 目的
建立一份比較 GPT 和 Claude 對《有機分類帽》遊戲的評估與改進建議，讓你可以快速檢視兩者之間的共識與差異。

---

## GPT 的觀察與建議

### 1. 直覺性與模式說明
- 按鈕、題目、選項要清楚，選單與模式切換需要讓玩家一眼懂。
- Practice / Speed / Duel 模式應該說明清楚，避免玩家誤以為所有模式都有倒數時間。

### 2. 答題反饋
- 答對時應增加更明顯的成功效果與動畫。
- 答錯時應標示正確答案，讓學習過程更有幫助。
- 目前錯誤懲罰只是 1 秒鎖定，要更有教學性。

### 3. 選項生成邏輯
- `generateOptions()` 應該從相似類別中抽干擾答案，避免完全無關題目。
- 一般關卡只要中文分子，level99 只要英文分類。
- 這樣可讓難度更穩定、題目更有針對性。

### 4. 對戰模式邏輯
- 現在 Duel 模式時間到直接平手，應改成比較分數/HP/答題數決勝。
- 這樣對決模式會更有張力。

### 5. 介面與規則清楚度
- Practice 模式應該隱藏時間條或改成專用樣式。
- 新增簡短規則說明區，讓玩家知道各模式玩法。
- 錯誤提示與 Combo 回饋應更清楚。

### 6. 邏輯健壯性
- `nextQuestion()` 若關卡資料不存在，應該顯示錯誤提示而不是 silent fail。
- `renderQuestion()` 目前只支援圖片題，若未來擴充題型應該更通用。
- 測試頁面可進一步檢查 `AnswerBank` 與 `QuestionSets` 一致性。

### 優先改進建議
1. 讓選項生成更合理
2. 答錯時顯示正確答案並強化反饋
3. 對決模式改成分數比較勝負
4. Practice 模式改成無時間提示

## Claude 的觀察與建議

### 1. 遊戲性
- 必須檢查答題回饋是否夠爽，包括音效、動畫、分數跳動。
- Combo 系統要有差異化視覺效果，讓玩家感受到進步。
- HP 條設計要平衡，答錯懲罰是否過重需測試。
- 競速模式和對決模式不能只成為手速比賽。

### 2. 教學邏輯
- 特別指出 `confused` 類別與官能基分類的問題：某些題目可能被錯誤歸類為「易混淆」，但實際上應該有正確官能基名稱。
- Level 6 與 Level 99 的分類對應要一致，否則學習者會混淆。
- Level 編排整體正確，但細節分類要修正。

### 3. 遊戲機制邏輯
- 抽題採 `Math.random()` 可能會重複出現相同題目。
- 干擾選項抽取過於隨機，缺少相似官能基干擾。
- Practice 模式目前沒有跑 timer，這點正常。
- 答錯 1 秒鎖定加扣 HP 20，連續錯誤可能令玩家挫折。

### 4. 一致性 / 邊界
- 對決模式雙方同時歸零時處理需確認。
- 對戰模式題目立即換掉是否會影響公平性。
- 缺少「題庫整輪不重複出題」機制。

### 5. 測試方法建議
- 找真實玩家測試 5 分鐘，觀察是否想再玩。
- 化學老師審查 `category` 欄位。
- 做全對 / 交錯答對答錯 / 全錯三種策略測試數值平衡。

### 優先改善方向
1. 修正分類邏輯不一致問題
2. 改進干擾選項品質
3. 加入抽題不重複機制
4. 增加重玩動機元素

## 結論
GPT 與 Claude 的建議高度一致：

- `分類邏輯一致性` 是首要修正項目
- `選項生成品質` 是提升好玩度的關鍵
- `答題回饋` 需要更強、更明確
- `對決模式` 目前設計可能會偏離遊戲本質

若你想進一步改進遊戲，建議先從「分類與題庫資料正確性」和「選項干擾品質」兩個方向著手，再把 UI/回饋與對戰規則作微調。

```javascript
document.addEventListener('keydown', e => {
    if (!gameActive) return;

        'a': 0, 'A': 0, '1': 0,
        'b': 1, 'B': 1, '2': 1,
        'c': 2, 'C': 2, '3': 2,
        'd': 3, 'D': 3, '4': 3

    if (btnIndex !== undefined) {
        const player = currentMode === 'duel' ? (e.shiftKey ? 'p2' : 'p1') : 'p1';
        const container = player === 'p1' ? UI.optsP1 : UI.optsP2;
        const btn = container.querySelectorAll('.opt-btn')[btnIndex];
        if (btn) btn.click();
    }
});
```

**用法**：
- P1：按 A/B/C/D 或 1/2/3/4
- P2（Duel 模式）：按 Shift+A/B/C/D
- 可選：ESC 鍵返回選單

| 改善項 | 具體做法 |
|---|---|
| **按鈕大小** | 桌面上增加按鈕尺寸至 60px+，目前可能太小；加上鍵盤提示標籤（「A」「B」「C」「D」） |

function detectInputMethod() {
