# 有機分類帽遊戲改進建議

## 1. 玩起來更直覺的方向
- 按鈕與模式說明要清楚：Practice / Speed / Duel 要讓玩家一眼懂。
- 手機橫放、雙人模式的操作應該更流暢。
- Practice 模式不應該讓玩家誤以為有倒數時間。

## 2. 增強答題反饋
- 答對時可增加更明顯的成功動畫或綠色閃爍。
- 答錯時應該短暫標示正確選項，讓玩家學習。
- 目前答錯僅鎖定 1 秒，改成顯示正確結果會更有教學效果。

## 3. 優化選項生成邏輯
- `generateOptions()` 應該從同一類別或同一關卡範圍選干擾答案，避免完全無關的干擾。
- 一般關卡應該只出中文化合物名稱，level99 應該只出英文官能基分類。
- 這樣可以讓難度更穩定、題目更有針對性。

## 4. 對戰模式改進
- 現在 `duel` 時間到會直接顯示平手，建議改成「比較分數 / HP / 答題數」決定勝負。
- 這樣對戰會更有張力，不會因時間到而顯得無聊。

## 5. 介面與遊戲規則清楚度
- Practice 模式應該隱藏或修改時間條樣式，避免混淆。
- 可以新增簡短規則說明區塊，例如：
  - Practice：無限練習
  - Speed：60 秒倒數挑戰
  - Duel：共享題目，分數決勝
- 錯誤提示與 Combo 回饋最好有更明確的文字與動畫。

## 6. 邏輯與健壯性建議
- `nextQuestion()` 若關卡資料不存在，應該提供錯誤提示或 fallback，而非直接失效。
- `renderQuestion()` 目前只支援圖片題，若未來要擴充其他題型，應該把題目渲染獨立成更通用的函式。
- 測試頁面可以進一步檢查：
  - 每個 `q.aKey` 是否存在 `AnswerBank`
  - 每個 `q.qContent` 是否能正常載入
  - `level99` 是否真的只出英文分類選項

## 7. 優先改進項目
1. 讓 `generateOptions()` 產生更合理、更同類的干擾選項。
2. 答錯時顯示正確答案並強化反饋。
3. Duel 模式時間到改成以分數/答題結果決勝。
4. Practice 模式改成無時間提示或專屬練習樣式。

## 總結
目前遊戲框架已經不錯，最需要改善的是「選項邏輯穩定性」與「答題回饋清楚度」。
如果這兩點做好，會讓遊戲更好玩、學習效果也更強。
